Initial Installation steps
The MySQL tasks below should be done via your web host's control panel, or via a SSH connection
1. Create an empty MySQL DB. The name can be anything you like, but is preferably "squirrelcart"
2. Create a MySQL User. The name and password can be anything you like
3. Add user to DB

The MySQL tasks below should be done via phpMyAdmin, which is offered by most web hosts in your control panel. If you do not have access to phpMyAdmin, you can obtain it at its homepage, at http://phpwizard.net.
4. Import DB by running MySQL query via the *.sql file included in the root folder of the Squirrelcart distribution. See image below for details:



5. Upload "squirrelcart" folder to the WWW root
6. Upload the contents of the images folder into the folder you store your images in. CHMOD each folder to 777:
a. content
b. categories
c. products
d. product_options
e. product_options\names
f. demo_store.jpg
g. sc_logo_80_percent.jpg
 7. Upload the "store.php" and "home.php" files into your web root.

a. $site_www_root = "http://www.mydomain.com";

Upgrading From a Previous Installation
Before performing an upgrade we HIGHLY recommend backing up your installation. For backup instructions, see this link.
The process below will temporarily take the store offline.

Important Note! - If you are upgrading from v1.1.0 to v1.2.0, click here for important information and refer to item #1.
Important Note! - If you are upgrading from v1.2.0 to v1.3.0, click here for important information.
1. Rename your existing "squirrelcart" folder to "squirrelcartOLD"
2. Follow steps 5, 7, and 8 in the initial installation instructions to upload the new squirrelcart folder. You can refer to your old config file for reference ("squirrelcartOLD/config.php").
3. If you see a new directory listed in step 6 of the installation instructions, then you must copy that directory to your images folder, and CHMOD it 777.
4. If you created a custom theme, then you must copy its folder from "squirrelcartOLD/themes" to "squirrelcart/themes".
Very important - DO NOT COPY THE ORIGINAL SQUIRRELCART THEME FOLDER INTO THE NEW INSTALL!!!!
5. To upgrade your DB, you must first determine what version of squirrelcart you are currently running. This should be listed in the "Release Notes.txt" file that came with your distribution. The version number also appears in your admin menu in v1.0.7 and newer.
6. Once you have determined what version you are upgrading from, download the corresponding upgrade file(s) from our support page - http://squirrelcart.com/support.php. The files are named with the version number at the beginning.
For example, the file "1.0.5_to_1.0.6_upgrade.sql" will upgrade your DB from v1.0.5 to v1.0.6. If you are more than 1 version behind, you will need to apply each upgrade file in ascending order.
Important Note! - If you are upgrading from v1.1.0 to v1.2.0, click here for important information and refer to item #2.
To apply the upgrade file to your DB, click on your database in phpMyAdmin, match the settings in the image below (for your upgrade file), and click "Go".

Templates and Themes
Customizing your cart's look and feel is done through the use of templates and themes.

Template definition: A Template is a file that controls a specific aspect of the cart. There are over 50 template files that control the look and feel of Squirrelcart. Each one is named according to the aspect of the cart that it controls. For example, "show_product.php" controls how a product is displayed.

Theme definition: A Theme is a set of template files designed to be used together to obtain a certain look and feel. Each theme has a unique name to describe it.

Changing the default theme: You can set a default theme for the cart using the "Default Theme" field on your store settings page. All customers that enter your store will see this theme by default.

Changing a user's theme: A customer can change their theme after they logon in the "Account Options" navigation box, without changing the default theme of your store. The cart will remember the theme they choose, and store it in their user record. Every time they log in, they will see the cart using the theme they have selected. To do this, they must first logon on your cart page, and then change the theme using the drop down box. If you are logged on as a Store Admin, and choose the "Change current theme..." link in the menu system, it will change the theme for the account that you are currently logged in with. It will not change the default theme.

Each theme has a record in the "Themes" table, and a folder within the "squirrelcart/themes" folder, as in this example:
squirrelcart/themes/brushed_steel
squirrelcart/themes/squirrelcart
squirrelcart/themes/supersquirrel
squirrelcart/themes/tigerframe

Important Note: Squirrelcart is the built in theme, and the templates within that folder should NEVER be modified!

Template Logic: When the cart needs a template in order to format content for your browser, it checks for that template file in the following locatons, in the order specified, stopping when it finds the template file it needs:
- the theme the user has specified (and is set in the user's record)
- the default theme specified in your store settings
- the Squirrelcart theme
Because of this, if you decide you do not like the template you modified you can delete it, and the cart will revert back to the built in template in the Squirrelcart theme.

Template Structure:
The majority of the content of a template file will be HTML. You can add any standard HTML tags to the templates to obtain the look you desire. Squirrelcart replaces the code that appears between the <? ?> symbols with data from the cart. For example, the show_product.php template looks like this:

<td height="190">
<form action="<?=$Form_Action?>" method="post">
<a href="<?=$Detail_Link?>"><img border="0" src="<?=$Thumbnail_Image?>"></a>
<br>
<div style="font-size: 8pt">
<?=$Name?>
<?=$Admin_Link?>
</div>
<a style="font-size: 7pt" href="<?=$Detail_Link?>">more detail...</a>
<div class="product_price" style="width: 100%">
<?=$Base_Price?>
</div>
<span style="position:relative; right: 20; top: 10; vertical-align:top"><?=$New?></span>
</form>
</td>

This template file is used to display a product when it's category is clicked on. Each variable between the <? ?> brackets represents a field in the Products table.

Modifying a Template:
1. Create your own theme in the Themes table, by following the link in the example image, and clicking "Add New Theme".
2. Create a new folder in the "squirrelcart/themes" folder with the same name as you specified in the "Path to Files" field on the theme's record.
3. If you want all customers to see the change you are going to make, set this new theme as your default theme on your store settings page.
4. Copy the template you wish to alter from "squirrelcart/themes/squirrelcart" into your new theme folder, and modify it with your HTML editor.

Important Note: DO NOT COPY ALL THE TEMPLATE FILES FROM THE SQUIRRELCART THEME FOLDER (squirrelcart/themes/squirrelcart) INTO YOUR CUSTOM THEME FOLDER!!!! (squirrelcart/themes/YourCustomTheme). This will cause many problems when upgrading your cart to a newer version.

Modifying Theme Images:
Cart images are also considered part of a theme. There are 2 ways to modify images for a particular theme.
Method 1:
Click the link in the menu system that reads "YOUR CUSTOM THEME Images" where YOUR CUSTOM THEME is the name of the theme you wish to modfy. This menu option will only take you to the images for the theme that is set for the account that you are currently logged in as. After clicking the link, you will see a table that lists all the images available for modification. Click on the image you wish to replace, browse to your new image file, and submit the record.
Method 2:
Upload a new image va FTP to your custom theme folder with a name that matches an existing image in the Squirrelcart theme.

Product Options
Product Options are used when you want to offer your customers choices that will modify the details of a particular item they are adding to their cart. There are 2 components. "Product Options", which defines the name of the option, such as size or color, and "Product Option Values", which defines the value of that option, such as large or blue. Here is how they work:

1. Create an option, and give it a name, such as "color".
This is done by clicking "Catalog > Products > Options" in the admin section. Then click "add new record", fill out the form, and click "submit record".

2. Then, create records for option values, and specify which option name they go with, like so:
Option Name: Color
Option Value: Red

Option Name: Color
Option Value: Green

Option Name: Color
Option Value: Yellow

Each one of the three examples above represents a record in the "Product Option Values" table, which you get to by clicking "Catalog > Products > Option Values". Do not confuse the word "values" in this context with monetary values. The word value represents the actual option choice you want to give the users.

3. Now, to add these options to a product, open a product record, and click "add option(s)" once for each choice you want to give the customer. You can also enter a 3 in the field next to the button, and click it once to add 3 items all at once to save some time.

4. On the product record, you should now see 3 options in the options section. You then set the Option Name for all three options to "Color". Then, set each option value as you want them to appear, red, green, yellow.

5. Now, if you want to charge extra for the option, you enter the price next to it. You can also enter a negative value, if you want the option to deduct from the base price of the item.

6. Click "submit" to submit the record changes

The cart adds the price of whatever options where chosen to the "Base Price" of the product. Here are 2 different ways you could use this, depending on your needs:


Example 1:
Say you are selling a 4 cylinder car, and it's Base Price is set to $10,000. Now, you want to offer a 6 cylinder option, and it costs $1500 more. Than you would set the Base Price to 10,000, and add an option for 6 cylinders that has a price of $1500. Now if a customer orders it and adds the 6 cylinder option, the total will be $11,500.

Example 2:
You may have a need to offer 1 item in different formats, at different prices. A good example is audio. Say you have a music store, and sell both tapes and CDs of the same album. You wouldn't want to set it up as in the first example, because in that case, there is no real base price. For something like this, you would leave the base Price as 0, and uncheck "Show Base Price". Then setup your options with different prices. Now when someone adds the item to the cart, they will only be charged for whatever option they selected, without a base price.


Shipping Options
There are 3 main components to the shipping functionality of Squirrelcart - Shipping Couriers, Shipping Methods, and Shipping Rules.

Shipping Couriers - This table contains a list of shipping couriers. If your store starting using a new courier, this is where you would add the name of the courier, and check "enable" to indicate that you are currently using it. The shipping methods that are explained below, are also listed on the shipping courier's record as "services". You can enable or disable a particular shipping method by checking or unchecking it on the courier's record.

Shipping Methods - This table contains a list of shipping methods. If you know that "Bob's Delivery" service offers both "Overnight" and "Super Express" delivery methods, you would enter each method here, as a separate record. The presence of a method in this table does not control whether or not it will be offered to the customer. The purpose of this table is to to keep track of the name of the method itself.
Shipping Rules - This table contains a list of rules that Squirrelcart uses to determine what shipping methods will be applied (or offered) to the customer. The table below lists the fields on each shipping rule record, and what they are used for:

For example, USPS Priority Mail is $3.50 for a small package. As the weight gets larger, the price goes up. You could program in certain rules based on your needs, and try and match them to whatever shipping method you are using. The "Priority" field is basically a tie breaker. It comes into play when you have 2 or more rules that apply for an order, as in this example:

Rule 1:
Apply when: Weight total is...
Operator: Less Than
Value: 1 (lbs - you don't need to enter lbs, just the number)
Shipping cost: $3.50
Apply to shipping type: USPS
Apply to shipping method: Priority Mail
Priority: 1

Rule 2:
Apply when: Weight total is...
Operator: Less Than
Value: 1 (lbs - you don't need to enter lbs, just the number)
Shipping cost: $1.50
Apply to shipping type: USPS
Apply to shipping method: First Class Mail
Priority: 1
Now say the order for this example is 1/2 of a pound. If you read these rules, both of them apply. So, if the priorities are the same, the customer gets to choose the shipping method at checkout. Now, say you have a free shipping policy on orders over $100. And you have a rule for that as well, and someone orders a rare postage stamp that costs $150. Then in this case, the first two rules for weight would apply (because stamps are light!), as well as your free shipping rule. So the only way you could force the free shipping rule (which you would probably want to do) is to give it a higher priority. So you would make its priority 2.

Important note: If you have a Courier set to disabled, it will still appear in the rules for the field "Apply To Shipping Type". If you set this field to a courier that is disabled, it will not apply unless you enable the courier itself. Each individual rule can also be disabled if you need to, by unchecking the "Enabled" field..

Real Time Rate Lookups
Squirrelcart supports real time rate lookups for USPS and UPS. This feature works as follows:
1. Customer clicks on "continue" link to be taken to a form to choose shipping method
2. Your (merchant's) web server connects to courier's web site, and requests rates for each package in the order.
3. Courier's server responds to merchant's server with rate information.
4. Merchant's web server then presents the rates to the customer for them to choose in the shipping method form

To enable Real Time Rate lookups for USPS, do the following:
1. Register online to use USPS Web Tools at the following URL: http://www.uspsprioritymail.com/et_regcert.html
2. Once your registration is complete, you will receive a username and password to access USPS Web Tools.
3. In Squirrelcart, login as an admin, and click "edit records" to go to the admin section of the cart
4. Click "Setting > Shippings > Couriers" in the menu system
5. Click on the record for "U.S.Postal Service"
6. Make sure "Enabled" is checked
7. Check the "Real Time Rates" field
8. Enter your username and password provided by USPS in the "RTR Username" and "RTR Password" fields, respectively.
9. The "RTR URL" is the URL that the cart will use to obtain rates. Confirm that it is set to "http://production.shippingapis.com/ShippingAPI.dll".
10. Check the boxes next to the services you wish to offer

To enable Real Time Rate lookups for UPS, do the following:
1. In Squirrelcart, login as an admin, and click "edit records" to go to the admin section of the cart
2. Click "Setting > Shippings > Couriers" in the menu system
3. Click on the record for "United Parcel Service"
4. Make sure "Enabled" is checked
5. Click "Enable UPS OnLine Tools®"
6. Follow the prompts in the forms, which will take you through the registration process
7. Click "Next" on the "Registration Successful" screen, and you will be returned to the UPS Shipping Courier record
8. You should now see a message that reads "You are registered to use UPS OnLine Tools®"

Squirrelcart automatically treats rate information obtained via RTR lookups as Shipping Rules. This allows you to offer your customers a combination of real time rates and Shipping Rules. To set the priority that these RTR Shipping Rules will use, use the field "RTR Shipping Rule Priority" on the record for each shipping courier.

Real time rate information is provided on a per package basis. To handle this, Squirrelcart has a packing function that it uses to determine a rough estimate of the number of packages that are in the order. It then requests rate information for each package, totals the amount per package, and offers the total amount to the customer for the corresponding shipping method. Squirrelcart uses the value of the Max. Weight field on the Shipping Method's record to determine how to pack the order. This method is not foolproof, but does work very well. If you wish your shipping rates to be more accurate, we recommend you check the "Ship as a separate package" option on your product records. The cart will then assume that you are shipping every item ordered in a separate package.


UPS Tracking
If you follow the procedures for enabling UPS Rates and Services above, UPS Tracking will automaticlaly be enabled. This will appear in both the front end after login, and in the back end "Tools" menu.

Payment Options
Payment Methods

The Payment Methods table contains a list of the different methods of payment available to Squirrelcart. As of v1.1.0, these include Visa, MasterCard, American Express, Discover, Check or Money Order, Electronic Check, and PayPal.

To enable a particular payment method, click the checkbox next to it in the "Enabled" column.

Important Note: For the cart to use any of the the credit card payment methods, you will need to have access to a payment gateway system.

To accept checks or money orders via mail, you will need to:
Make sure you have filled out your company information in the top of your store settings page. This information will presented to the customer for them to mail their payment to.
To accept PayPal, you will need to:
1. obtain an account with PayPal.com
2. click on the record for PayPal in the Payment Methods table, and enter the email address associated with your PayPal account in the "PayPal Email" field.
To accept Electronic Checks (direct debit):
You will need to use Authorize.net's payment gateway system. As of v1.1.0, Squirrelcart only supports electronic checks via Authorize.net.
Payment Gateways
In order to accept payments via credit card, you will need to use a payment gateway system. To enable and configure a payment gateway:
1. Click on it's record in the Payment Gateways table
2. Fill out the fields on the record with the information provided by your payment gateway provider. The fields on the Payment Gateway records vary depending on the gateway. Many of the field names can be clicked on for help.
3. If you have the ability to put your payment gateway account, it is recommend that you do so. You can then test the checkout process. Squirrelcart will accept a credit card number of 4111111111111111 for testing purposes.
CVV2/CVC Credit Card Security Codes
To help reduce credit card fraud, credit card companies have begun putting a 3 or 4 digit security code on credit cards. It is designed to to help merchants verify that the customer has a legitimate card in hand at the time of the order. On most cards, the security code appears on the back of the card, in the signature panel, directly after the credit card number. On American Express, it appears on the front.


Card Verification Code for Visa, Mastercard, and Discover

Card Verification Code for American Express

Squirrelcart supports the collection of this security code. As of this writing, however, only the Authorize.net payment gateway module will pass the information along to the gateway URL as part of the transaction. The other payment gateway modules will be updated soon to support this feature.

To enable the collection of the CVV2/CVC security code, do the following:
1. Logon to your cart as admin
2. Click on "edit records"
3. Click on the store settings link
4. In the "Security Settings" section, check the box next to "Collect CVV2/CVC"
5. If you wish to make an order require that the security code be entered, check the box next to "CVV2/CVC is Required"

Inventory Control

Squirrelcart has the ability to keep track of the number of units you have in stock for any given product. To use this feature, do the following:
1. Logon to your cart as admin
2. Click on "edit records"
3. Click on the store settings link
4. Check the box next to "Use Inventory Control". You should then see 2 more fields that will appear directly below the checkbox, as shown in the image on the right.
5. Set the remaining fields, "Out of Stock Behavior" and "Out of Stock Indicator". For a detailed description of what these fields do, click on their names in the admin section of the cart.
6. Specify the number of units you have for each product in the "Number in Stock" field on your Product records.

The cart will then subtract from your inventory when someone adds an item to their cart. If they do not check out in the amount of time specified in the "Session Timeout" field on your store settings page, the items will be added back into your inventory. If the setting of your "Out of Stock Behavior" allows a customer to purchase an item even when it is out of stock, the "Number in Stock" field will become negative, to reflect the number sold since the point that your stock was depleted.
